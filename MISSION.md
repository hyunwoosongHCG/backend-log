# Mission: Rails 백엔드 개발자로 자립하기

## Why

프론트엔드 개발자로 일하다 백엔드 개발자로 직종을 전환했다. 현재 Performance Plus(Rails 7, Ruby, Grape API, MySQL) 프로젝트에서 실무 작업을 하고 있지만, MVC 구조, DB 개념, 인증/인가 등 백엔드 기초 없이 작업하고 있다. 실무 작업을 통해 개념을 연결하고, 혼자서도 기능을 설계하고 구현할 수 있는 수준이 되는 것이 목표다.

## Success looks like

- 새 기능 요구사항을 받았을 때, 마이그레이션 → 모델 → 컨트롤러 → 엔티티 순서를 스스로 설계할 수 있다
- 코드를 보고 "이 요청이 어떤 경로로 흘러가는지" 머릿속에 그릴 수 있다
- N+1 쿼리를 스스로 발견하고 수정할 수 있다
- RSpec 테스트를 보고 무엇을 검증하는지 읽을 수 있고, 새 테스트를 스스로 작성할 수 있다
- Sidekiq 워커를 이해하고 간단한 백그라운드 잡을 작성할 수 있다

## Constraints

- 강의를 듣는 것이 아니라 **실무 작업을 통해** 학습한다 — 레슨은 실무에서 마주친 코드를 중심으로 설계되어야 한다
- 순서대로 배울 수 없다 — 프로젝트 요구에 따라 순서가 바뀐다
- 프론트엔드(React, TypeScript) 지식은 충분하다 — 비유로 적극 활용할 것

## Out of scope

- 프론트엔드 개발 (이미 알고 있음)
- Rails 이외의 백엔드 프레임워크 (Django, Spring 등)
- DevOps, 인프라 (지금 단계에서는)
