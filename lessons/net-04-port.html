🎯 김영한 「모든 개발자를 위한 HTTP 웹 기본 지식」 · 인터넷 네트워크 — IP 주소만으로는 구분할 수 없는 "같은 컴퓨터 안 여러 앱"을 PORT가 어떻게 식별하는지 이해한다

핵심 한 줄

IP가 인터넷에서 내 컴퓨터(아파트 동)를 찾아주는 주소라면, PORT는 그 컴퓨터 안의 특정 애플리케이션(호수)을 식별하는 번호다. 같은 컴퓨터에서 여러 서버가 동시에 돌아갈 때, PORT가 각 앱의 문을 구분한다.

왜 PORT가 필요한가

앞 강의에서 IP의 세 번째 한계를 기억하는가? "같은 IP 서버에 여러 앱이 있으면 어느 앱으로 패킷을 보낼지 모른다"는 문제였다.

실제로 서버 한 대에는 여러 서비스가 동시에 돌아가는 경우가 많다. Rails 앱 서버, MySQL DB 서버, Redis 등 모두 같은 머신에 있을 수 있다. 목적지 IP만 알아서는 "Rails로 보낼지, DB로 보낼지" 구분이 불가능하다. 이걸 해결하는 것이 PORT다.

개발할 때 localhost:3000은 React 개발 서버, localhost:8080은 Rails API 서버, localhost:3306은 MySQL이다. IP(localhost = 127.0.0.1)는 같지만, 뒤의 번호 — 그것이 PORT다. PORT가 없었다면 같은 머신의 세 서버를 동시에 띄울 수 없었을 것이다.

PORT 번호 규칙

PORT 번호는 0 ~ 65535까지 할당할 수 있다. (16비트 정수)

범위 이름 설명
0 ~ 1023 Well-known PORT 국제 표준으로 이미 용도가 정해진 번호 — 임의로 쓰면 안 됨
1024 ~ 49151 Registered PORT 업체·단체가 등록해 쓰는 번호 (예: MySQL 3306, Redis 6379)
49152 ~ 65535 Dynamic PORT OS가 클라이언트 소켓에 임시로 할당하는 번호

Well-known PORT 주요 번호

PORT 프로토콜 용도
20, 21 FTP 파일 전송 프로토콜
23 TELNET 원격 접속 (암호화 없음, 현재 거의 미사용)
80 HTTP 웹 통신 (비암호화). http://example.com = example.com:80
443 HTTPS 암호화된 웹 통신. https://example.com = example.com:443
22 SSH 암호화된 원격 접속 (서버 배포 시 사용)

브라우저에서 https://example.com을 입력하면 PORT를 명시하지 않아도 브라우저가 자동으로 443을 붙인다. 즉, https://example.com:443과 동일하다. 프론트에서 http://localhost:3000처럼 항상 PORT를 명시했던 이유는 Well-known PORT가 아닌 번호를 쓰기 때문이다.

IP + PORT = 소켓 주소

IP와 PORT를 합치면 인터넷에서 특정 컴퓨터의 특정 앱을 정확하게 가리킬 수 있다. 이 조합을 소켓 주소(Socket Address)라 한다. 

192.168.0.103:3000   // IP:PORT — 이 컴퓨터(192.168.0.103)의 3000번 앱
142.250.196.46:443   // Google 서버의 HTTPS 서비스

TCP 패킷 헤더에는 출발 PORT + 목적지 PORT가 모두 담긴다. 서버는 목적지 PORT를 보고 "이 패킷은 Rails(3000)로, 저 패킷은 MySQL(3306)로" 라우팅한다.

정리

  • PORT = 한 컴퓨터(IP) 안에서 특정 앱을 구분하는 번호
  • 비유: IP = 아파트 동 번호, PORT = 호수
  • 범위: 0~65535. 0~1023은 Well-known(표준 예약)
  • 주요 번호: HTTP=80, HTTPS=443, FTP=20/21, SSH=22
  • IP + PORT = 소켓 주소 — 인터넷에서 특정 앱을 정확하게 가리킴
  • 평소 localhost:3000이 바로 이 개념을 쓰고 있던 것
