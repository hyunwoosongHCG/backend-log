레슨 2 — 상속 vs 믹스인, 그리고 Grape mount

2026-06-29 실무 출처: ppback V2::Comments helpers 리팩토링 이전: 레슨 1 — ? ! &.
check_one_on_one_setting 안에서 workspace를 바로 쓸 수 있는 이유가 뭔지 파악이 안 됐다. 상속인지, include인지, mount인지 — 세 가지가 각각 다른 개념이다.

1. 세 가지는 완전히 다른 것이다

class V2::Base < Grape::API   # (1) 클래스 상속
  include ::GrapeBase          # (2) 모듈 믹스인
  mount V2::Comments           # (3) Grape 마운트
end

한 줄에 세 개가 다 나온다. 하나씩 뜯어보자.

2. 클래스 상속 <

Ruby는 단일 상속이다. 부모 클래스는 딱 하나만 가질 수 있다. 상속하면 부모의 모든 것 — 메서드, 동작 방식, "정체성"까지 물려받는다. 

class V2::Base < Grape::API
# V2::Base는 Grape::API의 일종(is-a)
# version, format, prefix, resource, mount 등 Grape 기능을 전부 사용 가능
end
React에서 class MyComponent extends React.Component와 같다. 상속하는 순간 setState, render, 라이프사이클 메서드가 전부 딸려온다.

3. 모듈 믹스인 include

Ruby에서 모듈(Module)은 메서드 묶음이다. 클래스가 아니라서 직접 인스턴스를 만들 수 없고, include로 다른 클래스에 섞어 넣는 용도로 쓴다. 클래스 상속과 달리 여러 모듈을 동시에 포함할 수 있다. 

# GrapeBase는 모듈
module GrapeBase
  extend ActiveSupport::Concern
  included do
    helpers do
      def workspace ...   # workspace, current_user 등 정의
      def current_user ...
    end
  end
end

# V2::Base가 GrapeBase를 포함 → 메서드를 빌려옴
class V2::Base < Grape::API
  include ::GrapeBase   # GrapeBase 기능 추가
  include AnotherModule # 이것도 동시에 가능 (상속과 다른 점)
end
React Hook과 비슷하다. useAuth(), useFetch()를 여러 컴포넌트에 동시에 꽂을 수 있는 것처럼, 믹스인도 GrapeBase를 여러 클래스에 동시에 포함할 수 있다. 상속 관계가 생기는 게 아니라 기능만 빌려오는 것이다.

상속 vs 믹스인 핵심 차이

클래스 상속 < 모듈 믹스인 include
개수 제한 하나만 가능 여러 개 가능
관계 is-a (정체성 공유) has-behavior (기능만 빌림)
JS 대응 extends Custom Hook
ppback 예시 V2::Base < Grape::API include GrapeBase

4. Grape mount — 라우트 조합

mount는 Ruby 언어 기능이 아니다. Grape 프레임워크가 제공하는 메커니즘으로, 한 API의 라우트를 다른 API 아래에 붙인다. 

class V2::Base < Grape::API
  include ::GrapeBase   # workspace, current_user 여기서 정의

  resource 'workspaces/:workspace_id' do
    mount V2::Comments  # V2::Comments의 라우트를 여기에 붙임
  end
end

class V2::Comments < Grape::API  # Grape::API만 상속, GrapeBase include 없음
  helpers do
    def check_one_on_one_setting
      setting = workspace.one_on_one_setting  # workspace를 쓸 수 있다?
    end
  end
end

V2::Comments는 GrapeBase를 직접 include하지 않았는데 workspace를 쓸 수 있다. 이유는 Grape의 mount가 요청 컨텍스트를 전달하기 때문이다. 마운트된 자식 API는 부모의 helpers 컨텍스트 안에서 실행된다.

React Context와 비슷하다. 부모가 <Provider value={workspace}>로 값을 제공하면 어디에 중첩된 자식이든 useContext로 꺼내 쓸 수 있다. V2::Base가 Provider, workspace가 context 값, V2::Comments가 자식이다.

5. 전체 흐름 한눈에

① module GrapeBase — workspace, current_user 등 헬퍼 정의
↓ include
② class V2::Base < Grape::API — GrapeBase 믹스인, V2::Comments 마운트
↓ mount (요청 컨텍스트 전달)
③ class V2::Comments < Grape::API — workspace 직접 사용 가능
# 결론: 세 줄의 의미
class V2::Base < Grape::API   # Grape 기능 전체를 상속
  include ::GrapeBase          # workspace 등 헬퍼를 믹스인으로 추가
  mount V2::Comments           # 라우트 조합 + 컨텍스트 전달
end
