🎯 김영한 「모든 개발자를 위한 HTTP 웹 기본 지식」 · HTTP 상태코드 — PRG 패턴으로 새로고침 중복 제출을 막는 법과, 304 Not Modified로 캐시를 활용하는 법을 배운다

핵심 한 줄

PRG(Post/Redirect/Get) 패턴은 POST 결과를 GET 페이지로 리다이렉트해 새로고침 시 중복 제출을 원천 차단하고, 304 Not Modified는 "변한 게 없으면 캐시를 써"라는 신호로 불필요한 데이터 전송을 없앤다.

새로고침 중복 제출 문제

브라우저는 새로고침(F5)을 누르면 마지막 HTTP 요청을 재전송한다. 만약 마지막 요청이 POST /orders(주문 생성)였다면, 새로고침마다 주문이 또 생성된다.

사용자 → POST /orders (주문 생성)
↓
서버 → 200 OK + 주문 완료 HTML
↓
사용자 새로고침 → 브라우저가 POST /orders 재전송 ⚠️
↓
서버 → 주문 또 생성됨 (중복!)

PRG(Post/Redirect/Get) 패턴

POST 처리 후 3xx로 리다이렉트하면, 마지막 요청이 GET이 된다. 이후 새로고침해도 GET만 재전송되므로 중복 제출이 일어나지 않는다.

사용자 → POST /orders (주문 생성)
↓
서버 → 302 Found (또는 303) + Location: /orders/123
↓
브라우저 → GET /orders/123 자동 요청
↓
서버 → 200 OK + 주문 상세 HTML
↓
사용자 새로고침 → GET /orders/123 재전송 (안전)
React에서 axios.post('/orders') 후 router.push('/orders/123')로 이동하는 패턴이 바로 PRG다. 클라이언트 라우팅으로 구현하면 실제로 HTTP 리다이렉션이 일어나지 않지만, "POST 결과를 GET 페이지로 보낸다"는 의도는 동일하다. 전통적인 MPA(Multi-Page Application)에서는 서버가 3xx로 PRG를 직접 처리한다.

Rails에서 PRG 구현

# Rails MPA 방식: PRG 패턴
class OrdersController < ApplicationController
  def create
    order = Order.create!(order_params)
    # POST 완료 → GET 상세 페이지로 리다이렉트 (PRG)
    redirect_to order_path(order), status: :see_other  # 303
  end
end

# API 모드(SPA)라면 201 Created로 응답하고,
# 프론트 JS가 router.push()로 이동한다.

302와 303 중 어느 것을 써도 대부분 브라우저에서 GET으로 변환되지만, 303 See Other는 "GET으로 바꾸라"는 의미를 명시적으로 담고 있어 PRG 패턴에 의미상 더 정확하다.

304 Not Modified — 캐시 재사용

304는 리다이렉션 계열(3xx)이지만 "다른 주소로 이동"이 아니라 "리소스가 변경되지 않았으니 캐시를 써"라는 캐시 목적 코드다.

동작 원리

브라우저는 서버로부터 리소스를 받을 때 ETag나 Last-Modified를 함께 저장한다. 다음 요청 시 If-None-Match 또는 If-Modified-Since 헤더에 이 값을 담아 보낸다. 서버가 확인했을 때 리소스가 변경되지 않았으면 304를 반환하고 응답 바디는 보내지 않는다. 브라우저는 캐시에 저장된 데이터를 그대로 사용한다.

첫 번째 요청 → GET /api/products
↓
서버 → 200 OK + 바디 + ETag: "abc123"
↓
두 번째 요청 → GET /api/products + If-None-Match: "abc123"
↓
서버 → 리소스 미변경 확인 → 304 Not Modified (바디 없음)
↓
브라우저 → 캐시 데이터 사용 (네트워크 절약)

304 응답에는 메시지 바디가 없다. 바디를 실어 보내면 캐시 절약이 의미가 없다.

Rails에서 조건부 GET

# Rails: 304 Not Modified 자동 처리
def show
  @product = Product.find(params[:id])
  # updated_at 기반으로 304 자동 판단
  if stale?(@product)
    render json: @product  # 변경됐으면 200 + 바디
  end
  # stale?가 false이면 Rails가 자동으로 304 반환
end

3xx 전체 정리

코드 종류 메서드 주요 용도
301 영구 GET으로 변경 가능 도메인·경로 영구 이전
308 영구 유지 메서드 보장하며 영구 이전
302 일시 GET으로 변경 가능 일반적인 일시 리다이렉트
307 일시 유지 메서드 보장하며 일시 이동
303 일시 GET으로 명시적 변경 PRG 패턴
304 캐시 - 변경 없음, 캐시 사용
