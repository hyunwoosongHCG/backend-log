핵심 한 줄

3xx 리다이렉션(Redirection)은 "이 주소는 지금 여기가 아니야, Location 헤더에 적은 주소로 다시 요청해"라는 신호다 — 영구 이동(301·308)이냐 일시 이동(302·307·303)이냐에 따라 검색엔진·브라우저·HTTP 메서드의 동작이 달라진다.

리다이렉션이란

브라우저가 서버에 요청을 보냈는데 서버가 3xx를 응답하면, 브라우저는 응답 헤더의 Location 값을 꺼내 자동으로 그 주소에 다시 요청을 보낸다. 이 동작 전체를 리다이렉션이라고 부른다.

클라이언트 → GET /old-url 요청
↓
서버 → 301 Moved Permanently + Location: /new-url
↓
클라이언트 → GET /new-url 자동 재요청
↓
서버 → 200 OK + 응답 바디
React Router에서 <Navigate to="/new-path" />로 특정 경로를 다른 경로로 보내는 것과 같은 개념이다. 다른 점은 HTTP 리다이렉션은 서버가 브라우저에게 명령한다는 것이고, 클라이언트 라우팅은 JS가 직접 URL을 바꾼다는 것이다.

영구 리다이렉션 (Permanent Redirection)

리소스 URI가 영구적으로 이동했을 때 쓴다. 검색엔진은 이 신호를 받으면 인덱스를 새 주소로 업데이트한다.

코드 이름 메서드 변경 바디 유지
301 Moved Permanently GET으로 변경될 수 있음 변경될 수 있음
308 Permanent Redirect 유지 (POST → POST) 유지

301 Moved Permanently

리다이렉트 후 HTTP 메서드가 GET으로 변경될 수 있다. 즉 POST /old-url을 보냈을 때 301을 받으면, 브라우저는 GET /new-url로 재요청할 수 있다. 원래는 메서드를 유지해야 했지만 역사적으로 브라우저들이 GET으로 바꾸는 방향으로 구현했고, 지금은 이 동작이 사실상 표준이 됐다.

308 Permanent Redirect

301과 기능은 같지만 메서드와 바디를 그대로 유지한다. POST /old-url을 보내 308을 받으면 POST /new-url로 재요청한다.

일시 리다이렉션 (Temporary Redirection)

리소스 URI가 일시적으로 다른 곳에 있거나, 특정 조건에서만 다른 주소로 보내야 할 때 쓴다. 검색엔진은 기존 주소를 그대로 유지한다.

코드 이름 메서드 변경 바디 유지
302 Found GET으로 변경될 수 있음 변경될 수 있음
307 Temporary Redirect 유지 (POST → POST) 유지
303 See Other GET으로 변경 (명시적) 변경

302 Found

가장 흔히 쓰이는 일시 리다이렉션 코드다. 301과 마찬가지로 메서드가 GET으로 바뀔 수 있다. 이 "의도치 않은 GET 변환" 문제를 해결하기 위해 307과 303이 등장했다.

307 Temporary Redirect

302와 같지만 메서드와 바디를 반드시 유지해야 한다. POST로 보낸 것은 POST로 재요청한다.

303 See Other

302와 같지만 재요청 메서드를 명시적으로 GET으로 변경한다. "다른 곳에서 봐라(See Other)" — 결과 페이지를 GET으로 조회하도록 유도할 때 쓴다. PRG(Post/Redirect/Get) 패턴에서 핵심적으로 사용된다 (다음 레슨에서 자세히).

실무 연결 — Rails에서 리다이렉트

# Rails: 영구 리다이렉션 (구 경로 → 신 경로)
get '/old-path', to: redirect('/new-path', status: 301)

# Rails: 일시 리다이렉션 (로그인 후 원래 페이지로)
def require_login
  unless current_user
    redirect_to login_path, status: :found  # 302
  end
end

정리

  • 3xx = 브라우저가 Location 헤더를 보고 자동으로 재요청
  • 301/308: 영구 이동 — 검색엔진이 인덱스 갱신
  • 302/307/303: 일시 이동 — 검색엔진은 기존 주소 유지
  • 301·302는 메서드가 GET으로 바뀔 수 있음 / 307·308은 메서드 유지
  • 303은 명시적으로 GET 전환 — PRG 패턴에 활용
