GET 요청이 JSON이 되기까지

레슨 003 Grape API · 컨트롤러 · Entity ~15분
🎯 미션 연결: 코드를 보고 "이 요청이 어떤 경로로 흘러가는지" 머릿속에 그릴 수 있다 — MISSION.md

들어가기 전에

오늘 작업한 GET /api/v1/workspaces/:workspace_id/one_on_one_settings를 예시로 쓴다. 프론트에서 이 API를 호출하면 서버 안에서 무슨 일이 일어나는지 — 그 전체 경로를 한 번에 잡는다.

전체 흐름 한눈에

① 프론트 → GET /api/v1/workspaces/abc/one_on_one_settings
↓ Rails 라우터가 받아서 Grape로 넘김
② V1::Workspaces (Grape endpoint) — 인증 확인, DB 조회
↓ present 호출
③ Entities::Workspaces::OneOnOneSettingEntity — 모델 → JSON 변환
↓ Grape가 JSON으로 직렬화
④ 프론트 → { "data": { "use_required_template": false, … } }

4단계다. 각 단계가 하는 일을 하나씩 보자.

① 라우터: 어디로 보낼지 결정

React Router에서 /settings 경로를 <SettingsPage>에 연결하는 것처럼, Rails 라우터는 URL을 컨트롤러에 연결한다.

Grape에서는 resource 'one_on_one_settings' do 블록 안의 get do가 GET /one_on_one_settings에 자동으로 연결된다. 별도의 routes 파일을 건드리지 않아도 된다.

② Grape Endpoint: 인증 + DB 조회

실제 코드를 보자. 오늘 작업한 파일 그대로다.

# app/controllers/v1/workspaces.rb (일부)

resource 'one_on_one_settings' do
  get do
    one_on_one_setting = OneOnOneSetting.find_or_create_by!(workspace_id: workspace.id)
    #                    ↑ DB에서 찾거나 없으면 만든다

    present :data, one_on_one_setting, with: Entities::Workspaces::OneOnOneSettingEntity
    #       ↑ 키   ↑ 모델 객체          ↑ 어떤 Entity로 변환할지
  end
end
React의 useEffect 안에서 fetch('/api/...')를 호출하고 데이터를 state에 저장하는 것처럼, 이 get do 블록이 "요청이 왔을 때 실행되는 함수"다. find_or_create_by!는 fetch, present는 return이라고 생각하면 된다.

find_or_create_by!(workspace_id: workspace.id)의 의미:

  • find_or_create_by! → 조건에 맞는 레코드가 있으면 찾고, 없으면 만든다
  • workspace_id: workspace.id → WHERE workspace_id = ? 조건
  • 끝에 ! → 실패하면 에러를 던진다 (조용히 실패하지 않음)

참고: ActiveRecord — find_or_create_by (공식 문서)

③ Entity: 모델 → JSON 변환기

present가 호출되면 Grape는 Entity 클래스에게 "이 모델을 JSON으로 만들어줘"라고 시킨다. 

# app/controllers/entities/workspaces/one_on_one_setting_entity.rb

class Entities::Workspaces::OneOnOneSettingEntity < Entities::Workspaces::BaseSettingEntity
  expose :use_inactive_alert
  expose :use_privacy_comment
  expose :use_required_template  # ← 오늘 추가한 것
  # …
end
React에서 API 응답에서 필요한 필드만 골라 state에 담는 것처럼, expose는 "이 필드를 JSON 응답에 포함해라"는 선언이다. 여기 없으면 DB에 값이 있어도 응답에 나오지 않는다.

expose에 리스트된 필드만 JSON에 담긴다. 보안 측면에서도 중요하다 — 의도치 않은 필드가 노출되지 않는다.

참고: grape-entity 공식 README

④ 최종 응답

{
  "data": {
    "inactive_alert_term": 0,
    "use_inactive_alert": false,
    "use_privacy_comment": true,
    "use_required_template": false,
    …
  }
}

Entity에 expose한 필드들이 그대로 나온다. 오늘 expose :use_required_template를 추가했기 때문에 이 응답에 포함된 것이다.

전체 역할 요약

단계 파일 하는 일 프론트 비유
라우터 resource 'one_on_one_settings' URL → 코드 연결 React Router
Endpoint get do … end 인증, DB 조회 useEffect + fetch
Entity expose :field 모델 → JSON 변환 응답 필드 선택
