2xx 성공

네트워크 · HTTP 상태코드 강의 23 05:53
🎯 김영한 「모든 개발자를 위한 HTTP 웹 기본 지식」 · HTTP 상태코드 — 200·201·202·204 네 코드를 구분해 "성공의 종류"를 정확히 표현하는 법을 배운다

핵심 한 줄

2xx는 "서버가 요청을 정상 처리했다"는 뜻이지만, 어떻게 성공했는지에 따라 200·201·202·204로 세분화해서 클라이언트가 다음 행동을 정확히 결정할 수 있게 한다.

2xx 코드 한눈에 보기

코드 이름 언제 쓰나 응답 바디
200 OK GET 조회, PUT 수정 성공 등 일반적인 성공 있음
201 Created POST로 새 리소스 생성 성공 있음 + Location 헤더
202 Accepted 요청은 받았으나 아직 처리 안 됨 (배치 작업) 선택
204 No Content DELETE 삭제 성공 등 응답 바디가 필요 없는 경우 없음

200 OK — 가장 기본적인 성공

요청을 받아 정상적으로 처리했고, 결과를 응답 바디에 담아 돌려준다. GET 요청의 조회 결과, 로그인 성공 응답 등 대부분의 성공 응답이 여기에 해당한다. 

# Rails: GET /api/users → 200 OK
def index
  users = User.all
  render json: users, status: :ok   # :ok = 200
end

201 Created — 리소스 생성 성공

POST 등으로 새 리소스가 생성됐을 때 돌려준다. 응답 헤더의 Location에 새로 만들어진 리소스의 URI를 담는 것이 관례다. 

# Rails: POST /api/users → 201 Created
def create
  user = User.create!(user_params)
  render json: user, status: :created,
         location: api_user_url(user)  # Location 헤더 자동 세팅
end

프론트에서는 201 응답과 함께 오는 Location(또는 바디의 id)으로 상세 페이지로 바로 이동시키는 패턴을 자주 쓴다. 

// 프론트: 201 받으면 생성된 리소스 상세로 이동
const response = await axios.post('/api/users', formData);
if (response.status === 201) {
  const newUserId = response.data.id;
  router.push(`/users/${newUserId}`);
}

202 Accepted — 비동기 처리 수락

요청은 접수됐지만 처리가 완료되지 않은 상태를 뜻한다. 대용량 이메일 발송, Sidekiq 백그라운드 잡 큐잉처럼 결과가 나오기까지 시간이 걸리는 작업에 쓴다. 서버는 "일단 받았다"는 신호만 준다. 

# Rails + Sidekiq: 202 Accepted
def send_bulk_email
  BulkEmailJob.perform_later(params[:campaign_id])
  render json: { message: '발송 요청 접수' }, status: :accepted  # 202
end

204 No Content — 바디 없는 성공

요청은 성공했지만 클라이언트에게 돌려줄 내용이 없을 때 사용한다. DELETE 삭제, 좋아요 토글 같이 "됐어, 데이터는 필요 없어"인 경우다. 응답 바디가 없으므로 Content-Length: 0. 

# Rails: DELETE /api/users/:id → 204 No Content
def destroy
  User.find(params[:id]).destroy!
  head :no_content   # head = 바디 없이 상태코드만 반환, :no_content = 204
end
axios에서 204를 받으면 response.data가 빈 문자열이다. response.data.id를 읽으려다 에러가 나는 버그를 종종 만난다. 204라면 "데이터 없음"으로 분기해야 한다.

200 vs 201 — 언제 구분하나

200으로 통일해도 기술적으로는 동작하지만, 201을 쓰면 "새 리소스가 생겼고, Location에 URI가 있다"는 의미가 명확해진다. REST API 설계 컨벤션으로 POST 성공 = 201이 표준이다.

정리

  • 200 OK: 일반적인 성공, 응답 바디 있음
  • 201 Created: 생성 성공, Location 헤더로 새 리소스 URI 제공
  • 202 Accepted: 비동기 처리 수락, 아직 완료 안 됨
  • 204 No Content: 성공이지만 응답 바디 없음 (DELETE 등)
  • axios는 2xx면 모두 resolve — 세부 코드로 UI 동작을 분기한다
