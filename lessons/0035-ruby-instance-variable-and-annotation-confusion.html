Ruby @ — 어노테이션이 아니라 인스턴스 변수

실제로 본 코드

최근 커밋(시간복잡도 개선)에서 이런 메서드를 봤다:

def badge_stats
  @badge_stats ||= badgepool_group.transform_values do |badge_count|
    {
      median: badge_count.median(user_count),
      max: badge_count.max || 0,
      average: user_count.zero? ? 0 : (badge_count.sum.to_f / user_count).round,
    }
  end
end

@badge_stats의 @ 기호를 보고 Java의 @Override, Python의 @decorator 같은 "어노테이션(annotation)"이라고 생각했다. 하지만 Ruby에는 언어 차원의 데코레이터/어노테이션 문법 자체가 없다.

Ruby의 @는 변수 종류를 나타내는 접두사

Ruby는 변수 이름 앞에 붙는 기호로 스코프(범위)를 구분한다:

변수명        # 지역 변수 (local variable)
@변수명       # 인스턴스 변수 (instance variable)
@@변수명      # 클래스 변수 (class variable)
$변수명       # 전역 변수 (global variable)

@badge_stats는 이 객체(인스턴스) 하나에 속한 변수라는 뜻일 뿐, 메서드나 클래스에 특별한 행동을 부여하는 문법이 아니다.

Q1. Ruby에서 @badge_stats의 @가 의미하는 것은?
⚠️ Python/Java 배경에서 오면 "함수 위/변수 앞의 @"를 보고 어노테이션·데코레이터를 떠올리기 쉽지만, Ruby의 @는 완전히 다른 개념이다:

// Python: @가 함수를 감싸는 데코레이터
@property
def name(self): ...

// JS: this.속성명으로 인스턴스 속성 표현
this.badgeStats = this.badgeStats || computeStats();

# Ruby: @가 인스턴스 변수 접두사
@badge_stats ||= compute_stats
JS의 this.badgeStats와 가장 비슷한 역할이라고 생각하면 된다. this. 대신 @를 쓴다는 차이뿐이다.

이 코드에서의 실제 동작 — 메모이제이션

||=는 "왼쪽 값이 nil/false면 오른쪽을 계산해서 대입하고, 이미 값이 있으면 그대로 반환"하는 연산자다. 인스턴스 변수와 결합하면 메모이제이션(memoization) 패턴이 된다 — 같은 요청·같은 인스턴스 안에서 비싼 계산을 한 번만 수행하고 캐싱해서 재사용한다.

실제로 이 메서드는 리팩터링 전에 percentile_series, radar_indicator, radar_series 세 곳에서 각각 median 정렬(O(n log n))과 max 스캔을 중복 수행하던 것을, badge_stats 하나로 모아 렌더당 한 번만 계산하도록 바꾼 것이다.

핵심 요약

기호 이름 범위
변수명 지역 변수 정의된 메서드/블록 내부
