핵심 한 줄

좋은 API URI는 동작(행위)이 아니라 리소스(자원) 그 자체를 표현하고, 행위는 HTTP 메서드로 구분한다.

나쁜 URI vs 좋은 URI

처음 API를 설계할 때 흔히 이렇게 만든다: 

# 나쁜 예 — URI에 동작이 들어있다
GET  /get-members
POST /create-member
POST /modify-member/1
GET  /delete-member/1

이게 왜 문제일까? URI는 리소스(Resource)를 식별하는 주소다. "회원을 등록한다", "회원을 삭제한다"는 동작(행위)이지 리소스가 아니다. 리소스는 "회원"이라는 개념 자체다.

동작을 URI에서 빼고, 행위를 HTTP 메서드로 표현하면: 

# 좋은 예 — URI는 리소스, 행위는 메서드로
GET    /members        # 회원 목록 조회
GET    /members/{id}   # 특정 회원 조회
POST   /members        # 회원 등록
PUT    /members/{id}   # 회원 수정 (전체 대체)
PATCH  /members/{id}   # 회원 수정 (일부 변경)
DELETE /members/{id}   # 회원 삭제

리소스(Resource) 식별이란?

URI에서 가장 중요한 것은 리소스를 식별하는 것이다.

  • 리소스(Resource): 회원, 주문, 상품 — "무엇"에 해당하는 개념 그 자체
  • 행위(Behavior): 조회, 등록, 수정, 삭제 — "어떻게 할 것인가"

URI는 리소스만 표현한다. 행위는 HTTP 메서드(GET, POST, PUT, PATCH, DELETE)에 맡긴다.

컬렉션(Collection): 왜 복수형인가?

/member가 아니라 /members인 이유가 있다. API에서는 계층 구조의 상위 자원을 컬렉션(Collection)이라 부르고, 복수형을 사용하는 것이 관례다. 

/members       ← 회원 전체 컬렉션
/members/1     ← 컬렉션 중 id=1인 특정 회원

컨트롤 URI: 어쩔 수 없는 예외

완벽한 RESTful이 안 될 때가 있다. 예를 들어 "주문을 배달 시작 처리"는 어떤 리소스, 어떤 메서드로 표현해야 할까? 이럴 때는 컨트롤 URI(Control URI)를 사용한다 — 동사로 된 URI를 예외적으로 허용한다. 

POST /orders/{orderId}/start-delivery   # 배달 시작
POST /members/{id}/block                # 회원 차단
POST /payments/{id}/cancel              # 결제 취소

이는 "리소스 중심 URI" 원칙의 예외다. 어떤 메서드로도 행위를 표현할 수 없을 때, 컨트롤 URI로 해결하고 최대한 POST를 쓰는 것이 관례다.

React Router에서 경로를 이렇게 설계하지 않는가: /users, /users/:id, /users/:id/settings. 행위를 경로에 담지 않고 컴포넌트가 무엇인지를 표현하는 것과 같다. Rails의 resources :members 한 줄이 만들어주는 7개 라우트도 이 원칙 그대로다 — URI는 리소스, 행위는 HTTP 메서드.

Rails resources 라우트와 대조

Rails에서 resources :members를 선언하면 자동으로 생성되는 라우트:

HTTP 메서드 URI Rails 액션 의미
GET/membersindex목록 조회
GET/members/:idshow단건 조회
POST/memberscreate등록
GET/members/newnew등록 폼
GET/members/:id/editedit수정 폼
PATCH/PUT/members/:idupdate수정
DELETE/members/:iddestroy삭제

이 7개 라우트가 REST의 기본 패턴이다. URI는 리소스, 행위는 메서드로 분리되어 있다.

정리

  • URI 설계의 핵심 = 리소스(Resource) 식별. 동작은 URI에 담지 않는다
  • 행위는 HTTP 메서드(GET/POST/PUT/PATCH/DELETE)로 표현한다
  • 상위 자원은 복수형 컬렉션으로 표기: /member → /members
  • 메서드로 표현 불가능한 행위는 컨트롤 URI로 예외 처리 (동사 URI + POST)
