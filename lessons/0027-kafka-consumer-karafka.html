Consumer란

Consumer(컨슈머)는 토픽을 구독해서 메시지를 읽고 처리하는 쪽이다. Karafka는 Ruby에서 카프카 Consumer를 만들 때 쓰는 프레임워크 이름이다 (Rails의 ActionController가 HTTP 요청을 다루는 프레임워크이듯, Karafka는 카프카 메시지를 다루는 프레임워크라고 보면 된다).

theplus-back의 karafka.rb에는 토픽마다 어떤 Consumer 클래스가 처리할지 라우팅이 정의돼 있다: 

# theplus-back/karafka.rb
routes.draw do
  consumer_group :perpl do
    topic :'record-hr-organization-code' do
      consumer HrmOrganizationInfoConsumer
      dead_letter_queue(topic: 'record-hr-organization-code-fail', max_retries: 2)
    end
  end
end

그리고 실제 Consumer 클래스는 이렇게 생겼다:

# theplus-back/app/consumers/hrm_organization_info_consumer.rb
class HrmOrganizationInfoConsumer < ApplicationConsumer
  def consume
    messages.each do |message|
      case message.key
      when "Organization"
        Consume::HrmOrganizationConsume.call(message: message)
      end
      mark_as_consumed(message)
    end
  end
end
Karafka의 consume은 Rails 컨트롤러의 액션과 비슷하다 — 요청(메시지)을 받아서 반응하는 진입점이다. 다만 트리거가 HTTP 요청이 아니라 브로커를 계속 폴링(polling)하는 백그라운드 프로세스라는 점이 다르다.

프론트로 치면 useEffect 안의 socket.on('message', handler)에 가깝다. 사용자 클릭 같은 일회성 이벤트가 아니라, 앱이 켜져 있는 동안 계속 살아서 새 메시지를 기다리는 리스너다.
Q1. Karafka Consumer의 consume 메서드는 언제 호출되는가?

Consumer는 얇고, 실제 로직은 서비스 객체가 한다

위 코드에서 HrmOrganizationInfoConsumer는 메시지의 key를 보고 어디로 보낼지 "라우팅"만 한다. 실제 DB write는 Consume::HrmOrganizationConsume이라는 별도 서비스 객체가 담당한다. Grape 컨트롤러가 얇고 실제 로직은 서비스 레이어에 있는 것과 똑같은 패턴이다 — 진입점(Consumer)과 로직(Service)을 분리한다. 

# theplus-back/app/services/consume/hrm_organization_consume.rb (요약)
class Consume::HrmOrganizationConsume
  def self.call(message:)
    data = message.payload['data']
    workspace = Workspace.find_by(client_id: data['clientId'])
    organization = workspace.organizations.find_by(uid: "#{workspace.uid}_#{data['beforeOrganizationCode']}")
    organization.update!(uid: "#{workspace.uid}_#{data['afterOrganizationCode']}")
  end
end

존재하지 않는 Consumer — ppback의 경우

ppback도 Karafka와 karafka.rb를 갖고 있지만, 실제로 열어보면 라우팅 블록 전체가 주석 처리돼 있었다. app/consumers/에도 껍데기인 ApplicationConsumer만 있을 뿐, 구체적인 Consumer 클래스가 하나도 없다.

즉 "Karafka를 설치했다"와 "Consumer가 동작한다"는 완전히 다른 이야기다. ① 카프카 브로커에 연결되고, ② 특정 토픽을 구독하는 라우팅이 등록돼 있고, ③ 그 토픽을 처리할 Consumer 클래스가 있어야 비로소 "소비"가 일어난다. 셋 중 하나라도 빠지면 그냥 라이브러리만 설치된 상태다.

Q2. ppback에 Karafka gem이 설치돼 있고 karafka.rb도 있는데, 왜 "Consumer가 없다"고 하는가?

핵심 요약

개념 역할
Karafka Ruby용 카프카 Consumer 프레임워크 (Rails의 카프카 버전 진입점)
Consumer 클래스 consume 안에서 메시지를 받아 서비스 객체로 위임 + mark_as_consumed
"소비 중"의 조건 브로커 연결 + 라우팅 등록 + Consumer 클래스, 셋 다 있어야 한다
