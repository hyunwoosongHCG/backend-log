# backend-log

백엔드 개발자로 성장하는 과정을 기록하는 학습 포트폴리오.

실무 작업을 하면서 마주친 개념을 체크하고, 언제 어떤 작업에서 배웠는지 남긴다.

## 구조

```
backend-log/
├── ROADMAP.md          # 섹션별 체크리스트 ← 여기서 시작
├── concepts/           # 개념별 상세 정리
└── work-log/           # 어떤 작업에서 배웠는지
```

## 사용법

1. 작업을 하다가 새 개념을 마주치면 ROADMAP.md에서 찾는다
2. 체크박스를 `[x]`로 바꾸고, 개념 파일과 work-log 링크를 연결한다
3. 로드맵에 없는 개념이면 적절한 섹션에 항목을 추가한다

## 체크 형식

```markdown
- [x] 개념 이름 → [정리](concepts/개념.md) | [배운 작업](work-log/날짜-작업명.md)
- [ ] 아직 안 배운 개념
```
